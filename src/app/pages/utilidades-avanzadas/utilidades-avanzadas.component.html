Utilidades avanzadas

1.- Operadores throwError, carthError y retry

throwError es una función que se limita a crear un observable que lanza un error. Una vez que
se lanza el error, se cancela la suscripción.

Con catchError es posible intervenir en el error a nivel de observable. Es decir, antes de que se
emita la suscripción y se cierre. Esto nos permite evitar que se cancele la suscripción en caso de error.

El operador retry nos permite capturar una excepción en el flujo de datos y reintenter su ejecución
un número limitado de veces.

2.- constantes never y empty

never es una función que se devuelve un observable vacío. No emite valor, no lanza error ni se completa.

empty es una función que se devuelve un observable vacío que, tal y como se lanza, se completa.

3.- Eventos y notificaciones: materialize y dematerialize

Una notificación de RxJS es un objeto que representa una acción push de un observable

materialize nos permite emitir notificaciones de cada evento. Nos muestra varios parámetros:

  • kind es el tipo de evento. N: Next; C: Completed; E: Error.
  • hasValue indica si hay valor.
  • value muestra el valor (si lo hay).
  • error muestra el error (si lo hay).

dematerialize funciona al contrario: recibe eventos en forma de notificaciones y los transforma en
acciones sobre el propio observable.
