Operadores básicos

1.- Operadores mapTo, map y filter

[DEPRECADO] mapTo es el operador de transformación más simple que existe. Para cada evento que entra, transforma
la salida devolviendo siempre el mismo valor.

map aplica una función a cada valor emitido por el observable inicial y emite los valores resultantes
como un observable.

filter deja pasar solo los valores que cumplen una determinada condición. La función que le pasamos debe
devolver true o false.

2.- Operador tap

tap no modifica el flujo de datos; lo mismo que recibe es lo que emite. Pero lo que sí puede hacer
es ejecutar acciones ajenas al string de datos.

3.- Operadores first, take y takeWhile

Estos operadores sirven para limitar el número de eventos que recibimos de un stream.

first devuelve solamente el primer evento del stream. Pero, además, se le pueden pasar
unas condiciones concretas para dar por válido el evento, parecido a como lo haría filter.

El operador take sirve para indicar cuántos eventos queremos emitir antes de cerrar el stream.

takeWhile, en vez de emitir una cantidad concreta de eventos, emite eventos mientras se cumpla
una determinada condición. En cuanto deje de cumplirse dejará de emitir.

4.- Operadores last, takeLast y skip

Estos operadores sirven para limitar el número de eventos que recibimos de un stream.

last devuelve solamente el último evento del stream. Para saber que es el último, el stream
tiene que cerrarse

El operador takeLast sirve para emitir los últimos X eventos.

skip sirve para omitir los X primeros eventos.

5.- Operadores reduce y scan

reduce aplica una función a cada evento del stream de forma secuencial y solamente devuelve
el resultado cuando se cierra el stream. Esta función es el acumulador.

El operador scan es similar a reduce, pero emite un evento con el valor acumulado cada vez que
recibe un evento, sin esperar a que se cierre el stream.
